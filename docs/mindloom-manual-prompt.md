# Стандартный Mindloom Prompt для ручного workflow

## Зачем нужен этот prompt

До подключения GPT Action команда вручную загружает транскрипт / материалы в Mindloom GPT.
Чтобы каждый раз получать одинаковую JSON-структуру, которую backend может нормально обработать, нужна эта инструкция.

**Backend всё равно принимает любой JSON.** Prompt нужен только для стабильности ручного workflow.

---

## Ручной workflow (шаги)

1. Откройте Mindloom GPT в ChatGPT.
2. Скопируйте prompt-инструкцию ниже.
3. Вставьте инструкцию в чат + вставьте транскрипт / материалы сессии на место `[ВСТАВИТЬ ТРАНСКРИПТ / МАТЕРИАЛЫ СЕССИИ]`.
4. Получите от Mindloom чистый JSON.
5. Скопируйте JSON.
6. Отправьте через curl или Postman на `POST /api/mindloom/reports` (инструкция в [docs/testing.md](testing.md)).
7. Из ответа возьмите `report_url`.
8. Передайте ссылку участнику.

## Prompt-инструкция для Mindloom GPT

```
Проведи анализ по приведённым материалам.
Верни результат строго в формате JSON — без markdown, без пояснений до или после, без блоков ```json```.
Только чистый JSON-объект по следующей схеме:

{
  "participant": {
    "name": "имя участника или 'Участник' если не указано",
    "id": ""
  },
  "session": {
    "source": "mindloom_gpt",
    "date": "<текущая дата и время в формате ISO8601, например 2026-05-18T10:00:00Z>",
    "language": "ru",
    "version": "1.0"
  },
  "report": {
    "title": "название анализа",
    "summary": "краткое резюме в 2-4 предложениях",
    "blocks": [
      {
        "id": "уникальный_id_блока_латиницей",
        "title": "название блока",
        "text": "подробный текст анализа для этого блока",
        "quotes": ["точная цитата из материала, если есть"],
        "recommendations": ["конкретная рекомендация, если есть"]
      }
    ],
    "markers": [
      {
        "key": "ключ_маркера",
        "label": "Название маркера",
        "value": "значение (например: высокий / низкий / средний)"
      }
    ],
    "layers": []
  },
  "meta": {
    "schema_version": "1.0"
  }
}

Правила заполнения:
- blocks[] — минимум 3-5 смысловых блока, каждый со своим id и title
- markers[] — ключевые психологические маркеры из анализа (может быть пустым [])
- layers[] — оставить пустым []
- quotes[] — только прямые цитаты из материала, не пересказ
- recommendations[] — конкретные практические рекомендации или []
- Все текстовые поля на русском языке
- Никаких комментариев вне JSON

Материалы:
[ВСТАВИТЬ ТРАНСКРИПТ / МАТЕРИАЛЫ СЕССИИ]
```

## Важные правила

- Если Mindloom вернул текст вместо JSON — попросите повторить: `"Верни только JSON без пояснений"`
- Если JSON пришёл с ```json блоком — уберите обёртку вручную перед отправкой
- Backend сохранит любой JSON as-is, даже если структура отличается — отчёт будет показан как raw JSON
- Этот prompt — рекомендация, не жёсткий контракт
