# Mindloom Report v2 JSON Schema

Целевая схема для Mindloom Report v2.

`meta.schema_version` остаётся `"2.0"` — это эволюция v2, не новая версия.

---

## Концепция

Mindloom Report v2 — нейропсихологический аналитический отчёт по одному материалу или одной сессии.

Цель отчёта:

- быть глубоким и нейропсихологически точным — и понятным для обычного пользователя без специального образования
- сложные понятия объяснять простым языком прямо внутри полей `description`, `explanation`, `why_it_matters`
- не требовать отдельного глоссария или лекции
- содержать ключевые смысловые блоки: snapshot, phrase_microscope, honest_translation, protected_need, how_to_read

---

## Поля — новые смысловые блоки (Patch 4A)

### `snapshot` — Главное за 30 секунд

```json
{
  "key_pattern": "Усталость как плата за право быть ценным",
  "short_explanation": "Человек держит систему на гиперконтроле, потому что отдых переживается не как восстановление, а как риск потерять ценность.",
  "three_signals": [
    "Я должен тянуть всё сам",
    "Отдых вызывает вину",
    "Контроль создаёт ощущение безопасности"
  ],
  "main_overheat": {
    "label": "Гиперконтроль",
    "score": 0.91,
    "explanation": "Главная зона перегрева: постоянная необходимость всё проверять и удерживать."
  },
  "first_step": "Учиться выдерживать короткую паузу без немедленного возврата в контроль."
}
```

Правила:
- `key_pattern` — короткий, сильный, человеческий заголовок (не технический термин)
- `three_signals` — ровно 3 коротких проявления паттерна
- `score` — число от 0 до 1
- Не делать медицинских диагнозов
- Не придумывать факты вне материала

```json
{
  "title": "Как пользоваться отчётом",
  "steps": [
    {
      "title": "Сначала прочитайте главный паттерн",
      "description": "Он показывает центральную связку, вокруг которой собирается отчёт."
    },
    {
      "title": "Посмотрите карту перегрева",
      "description": "Она показывает, какие зоны системы сейчас активнее всего."
    },
    {
      "title": "Разберите фразу под микроскопом",
      "description": "Так видно, как паттерн проявляется прямо в речи."
    },
    {
      "title": "Выберите одну практику",
      "description": "Не пытайтесь исправить всё сразу — начните с одного действия."
```

### `phrase_microscope` — Фраза под микроскопом

```json
{
  "title": "Фраза под микроскопом",
  "quote": "Я не могу остановиться, иначе всё развалится",
  "why_this_quote": "Эта фраза показывает связку контроля, тревоги и ощущения личной ответственности за всю систему.",
  "fragments": [
    {
      "text": "не могу остановиться",
      "meaning": "контроль как защита",
      "pattern": "гиперконтроль",
      "explanation": "Остановка переживается не как отдых, а как потеря управления — поэтому она невозможна."
    },
    {
      "text": "иначе всё развалится",
      "meaning": "катастрофизация",
      "pattern": "страх потери контроля",
      "explanation": "Система ожидает резкого ухудшения, если контроль ослабнет хотя бы на время."
    }
  ],
  "summary": "Фраза показывает, что пауза воспринимается как угроза, а не как восстановление."
}
```

Правила:
- `quote` — реальная цитата из материала, не пересказ
- `fragments[].text` — фрагменты той же цитаты
- Если точной фразы нет — использовать самую близкую и указать это в `why_this_quote`
- Не выдумывать цитаты

### `honest_translation` — Как сказано / как честно

```json
{
  "title": "Как сказано / как честно",
  "items": [
    {
      "as_said": "Я просто слишком чувствительная",
      "more_honest": "Я чувствую глубоко, и мне нужна опора.",
      "explanation": "Первичная формулировка обесценивает чувствительность, а честная версия возвращает право на поддержку."
    },
    {
      "as_said": "Я не хочу никого нагружать",
      "more_honest": "Мне нужна поддержка, и я имею право её попросить.",
      "explanation": "За отказом от просьбы может стоять страх стать обузой."
    }
  ]
}
```

Правила:
- `as_said` — желательно из материала или близко к формулировке участника
- `more_honest` — бережный перевод смысла, не фантазия, не категоричный
- Не делать слишком жёстких утверждений

### `protected_need` — Потребность под защитой

```json
{
  "title": "Потребность под защитой",
  "description": "Одна потребность с трёх сторон: что названо в тексте, что стратегия пытается добыть и что приносится в жертву.",
  "named": ["надёжность связи", "автономия"],
  "strategy_gets": ["безопасность", "предсказуемость"],
  "sacrificed": ["отдых", "ясность", "уважение к себе"],
  "leading_need": "надёжность связи",
  "interpretation": "Ведущая потребность связана с безопасной опорой, но добывается через контроль и самонапряжение."
}
```

Правила:
- Это психологическая гипотеза, не диагноз
- Формулировать бережно
- Лучше меньше элементов, но точнее

### `feedback_config` (опционально)

```json
{
  "enabled": false,
  "positive_label": "Это про меня",
  "negative_label": "Не похоже на меня"
}
```

## Safety rules

- Это не медицинский диагноз
- Все выводы — гипотезы
- Evidence должно происходить из исходного материала
- Нельзя придумывать факты, которых нет в материале
- Все числовые поля `intensity`, `confidence`, `strength`, `score`, `overheat_level`, `resource_level` — в диапазоне `0..1`
- `phrase_microscope.quote` — реальная цитата из материала
- `honest_translation.more_honest` — бережный перевод, не категоричное утверждение

## Backward compatibility

Старые v2-отчёты без новых полей (`snapshot`, `how_to_read` и т.д.) продолжают работать — эти поля optional в normalizer. Renderer не упадёт на старых отчётах.

Патч 4B/4C добавит UI для новых блоков в публичный отчёт.
