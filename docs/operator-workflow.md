# Рабочий процесс оператора Mindloom

Пошаговая инструкция для обработки входящих заявок от клиентов.

---

## Mindloom Report v2 (Patch 4A — расширенная schema)

Текущий операторский workflow не меняется по шагам, но начиная с Patch 4A prompt для Mindloom включает новые смысловые блоки.

- В блоке **Шаг 1** система генерирует prompt для Report v2 (включая новые поля).
- Mindloom GPT должен вернуть результат в одном ` ```json ` code block.
- Внутри JSON должен быть `meta.schema_version: "2.0"`.
- В JSON не должно быть top-level поля `protocol`.
- Блок "что делать" должен жить внутри `recommended_practices`.
- Код-блок можно вставлять в форму целиком: система автоматически убирает markdown fences.
- Если Mindloom вернул старый schema v1 или `mindloom_report_v2_fixed_blocks`, использовать repair prompt или перегенерировать отчёт.

### Новые поля v2 (Patch 4A) — проверить при QA

Перед отправкой ссылки клиенту оператор должен убедиться, что в JSON присутствуют:

| Поле | Что это | Что проверить |
|---|---|---|
| `snapshot` | Главное за 30 секунд | `key_pattern` — короткий человеческий заголовок; `three_signals` — 3 сигнала |
| `phrase_microscope` | Фраза под микроскопом | `quote` — реальная цитата из материала (не пересказ) |
| `honest_translation` | Как сказано / как честно | `items` не пустой; `as_said` из материала |
| `protected_need` | Потребность под защитой | `named`, `strategy_gets`, `sacrificed` заполнены |
| `how_to_read` | Как пользоваться отчётом | `steps` содержит 3–4 шага |
| `heatmap.callouts` | Пояснения к карте | хотя бы 1–2 callout |
| `node_graph.legend` | Легенда графа | 4–5 типов связей |
| `node_graph.central_node_id` | Ключевой узел | совпадает с `id` одного из `nodes` |

## Основной цикл

### 1. Открыть Custom GPT

Клиент отправляет материалы (транскрипт, текст, описание ситуации) через Custom GPT в ChatGPT.  
GPT автоматически создаёт заявку через GPT Action → заявка появляется в `/admin/intake`.

---

### 2. Открыть список заявок

Перейдите на:
```
https://mindloom.edagency.ru/admin/intake
```

Введите логин и пароль (HTTP Basic Auth).  
Найдите новую заявку со статусом **новая**.

---

### 3. Открыть заявку

Нажмите **Подробнее →** рядом с нужной заявкой.  
Откроется страница заявки с:
- материалом клиента (текст)
- блоком **Шаг 1 — Скопировать промпт для Mindloom v2**
- формой **Шаг 2 — Вставить JSON отчёта v2 от Mindloom**

---

### 4. Скопировать промпт

В блоке **Шаг 1** нажмите кнопку **Скопировать промпт v2**.  
В буфер обмена попадёт готовый промпт с материалом клиента.

---

### 5. Вставить промпт в Mindloom GPT

Откройте **Mindloom GPT** в ChatGPT.  
Вставьте скопированный промпт.  
Mindloom GPT должен вернуть один ` ```json ` code block с JSON-отчётом v2 и без лишнего текста.

---

### 6. Скопировать JSON от Mindloom

Mindloom GPT вернёт ответ в блоке кода ` ```json `.  
Нажмите кнопку **Copy** у блока кода (или выделите его целиком и скопируйте).  
Можно копировать вместе с ` ```json ` — форма уберёт их автоматически.  
Если нет кнопки Copy — выделите от первой `{` до последней `}` и скопируйте.

---

### 7. Вставить JSON в форму

Вернитесь на страницу заявки в `/admin/intake/:id`.  
В блоке **Шаг 2** вставьте JSON в текстовое поле.

---

### 8. Нажать «Создать отчёт»

Нажмите кнопку **Создать отчёт**.  
Через секунду появится сообщение **Отчёт создан. Заявка завершена.** и публичная ссылка на отчёт.

---

### 9. Проверить отчёт

Нажмите **Открыть отчёт ↗** — откроется публичная страница отчёта.  
Проверьте, что всё отображается корректно:
- заголовок и имя участника
- краткое резюме
- маркеры
- блоки анализа с цитатами и рекомендациями
- уровни анализа

Все блоки Patch 4A (`snapshot`, `phrase_microscope`, `honest_translation`, `protected_need`, `how_to_read`, расширенные `heatmap`/`node_graph`) отображаются в ReportV2Dashboard начиная с Patch 4B–4N. Интерактивные нижние листы (bottom sheets) добавлены в Patch 4M.

---

### 10. Скопировать ссылку

Нажмите **Скопировать ссылку**, чтобы скопировать публичный URL отчёта.  
Ссылка открывается без авторизации — её можно отправлять клиенту напрямую.

---

### 11. Отправить ссылку клиенту

Передайте ссылку клиенту удобным способом (мессенджер, email и т.д.).  
Страница отчёта открывается на любом устройстве — никаких аккаунтов не требуется.

---

## Если ошибка

### «JSON невалидный» при создании отчёта

Форма автоматически убирает ` ```json ` обёртку из вставленного текста.  
Если JSON всё равно невалидный:
- убедитесь, что нет лишнего текста до или после JSON-объекта;
- ответ должен начинаться с `{` и заканчиваться `}`;
- структура должна соответствовать Mindloom Report v2 (`meta.schema_version: "2.0"`, `active_nodes`, `heatmap`, `node_graph`, `recommended_practices` и остальные top-level поля);
- JSON не должен содержать top-level поле `protocol`.

---

### Нет доступа к админке (401)

Браузер запросил логин/пароль — введите `ADMIN_USERNAME` / `ADMIN_PASSWORD` из `.env`.  
Если забыли пароль — уточните у администратора сервера.

---

### Отчёт уже создан (409)

Если нажать «Создать отчёт» повторно, форма покажет: «Отчёт для этой заявки уже создан» и отобразит ссылку на существующий отчёт.  
Ничего страшного — используйте ссылку из этого сообщения.

---

### Заявка не появилась в /admin/intake

Возможные причины:
- GPT Action не отработал — попросите клиента повторить отправку
- Посмотрите `/api/health` — если `db: "error"`, нужна перезагрузка сервиса
- Проверьте логи: `docker logs mindloom-receiver --tail 50`

---

### Custom GPT Action не сработал

- Убедитесь, что GPT Action активен в GPT Builder (Actions → submitMindloomIntake)
- Проверьте, что Bearer token в GPT Builder совпадает с `MINDLOOM_WEBHOOK_SECRET` на сервере
- Попробуйте тест прямо в GPT Builder: Actions → Test → submitMindloomIntake

---

## Важно про TXT-транскрипты

Для TXT-транскриптов в заявке должен быть **полный текст разговора слово в слово**.

**Правильная заявка содержит:**
- прямую речь и реплики участников;
- повторы, паузы, исходные формулировки;
- timestamps и speaker labels, если они были в файле.

**Неправильная заявка содержит:**
- «основные темы» или «краткое содержание»;
- «структурированная расшифровка» или «структурированный конспект»;
- «доступная часть материала»;
- `extracted_text_from_large_file`;
- пересказ вместо полного текста.

**Если оператор видит summary вместо полного разговора:**
1. Не отправлять такую заявку в Mindloom.
2. Попросить пользователя разделить транскрипт на части или отправить полный текст вручную.
3. Создать intake заново с полным текстом.

## Ручной способ (если GPT Action недоступен)

Если Custom GPT Action временно не работает, можно создать intake вручную через `npm run submit:intake` (см. [docs/testing.md](testing.md)), либо отправить отчёт напрямую через `npm run submit:sample`.
