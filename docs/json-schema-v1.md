# Mindloom JSON Schema v1

Описание стандартной JSON-схемы для отчётов Mindloom.

---

## Принцип работы

**Endpoint `/api/mindloom/reports` принимает любой валидный JSON** — строгой схемы нет.  
Если JSON соответствует schema v1, публичная страница `/r/:token` отображает красиво структурированный отчёт.  
Если структура отличается — страница показывает безопасный fallback без raw JSON.  
Raw JSON доступен только в `/admin/reports/:id`.

---

## Канонические правила schema v1

- **`meta.schema_version`** должен быть `"1.0"` — поле находится внутри объекта `meta`, не на верхнем уровне.
- **`schema_version: "mindloom_report_v2_fixed_blocks"`** на верхнем уровне — запрещён. Это другой, устаревший формат.
- **`session`** — обязателен. Поле `source` на верхнем уровне (вместо `session`) не поддерживается.
- **`session.source`** должен быть `"mindloom_gpt"`.
- **`report.blocks`** должен быть **массивом** (`[]`), не объектом (`{}`).
- **`report.markers`** должен быть массивом строк (`string[]`).
- **`report.layers`** должен быть массивом объектов `{ "name": "...", "description": "..." }`.

---

## Полная структура schema v1

```json
{
  "participant": {
    "name": "Имя участника"
  },
  "session": {
    "source": "mindloom_custom_gpt",
    "date": "2026-05-19T10:00:00Z",
    "language": "ru",
    "version": "1.0"
  },
  "report": {
    "title": "Название анализа",
    "summary": "Краткое резюме в 2–4 предложениях.",
    "blocks": [
      {
        "id": "block_id_latinitsa",
        "title": "Название блока",
        "text": "Подробный текст анализа для этого блока.",
        "quotes": [
          "Точная цитата из материала"
        ],
        "recommendations": [
          "Конкретная рекомендация"
        ]
      }
    ],
    "markers": [
      "маркер как строка",
      "другой маркер"
    ],
    "layers": [
      {
        "name": "Название уровня",
        "description": "Описание уровня анализа."
      }
    ]
  },
  "meta": {
    "schema_version": "1.0"
  }
}
```

---

## Поля

### `participant`

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `participant.name` | string | нет | Имя или псевдоним участника |

---

### `session`

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `session.source` | string | нет | Источник заявки: `mindloom_custom_gpt`, `mindloom_gpt` и др. |
| `session.date` | string (ISO8601) | нет | Дата сессии |
| `session.language` | string | нет | Язык: `ru`, `en` и т.д. |
| `session.version` | string | нет | Версия промпта: `"1.0"` |

---

### `report`

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `report.title` | string | нет | Заголовок отчёта (отображается на публичной странице) |
| `report.summary` | string | нет | Краткое резюме |
| `report.blocks` | array | нет | Блоки анализа (см. ниже) |
| `report.markers` | array | нет | Ключевые маркеры — массив строк или объектов |
| `report.layers` | array | нет | Уровни анализа Mindloom (см. ниже) |

---

### `report.blocks[]`

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Уникальный идентификатор блока (латиница) |
| `title` | string | Заголовок блока |
| `text` | string | Основной текст анализа |
| `quotes` | string[] | Прямые цитаты из материала клиента |
| `recommendations` | string[] | Конкретные рекомендации |

---

### `report.markers[]`

Markers поддерживают два формата:

**Строки (рекомендуется):**
```json
"markers": [
  "хроническая усталость",
  "гиперответственность",
  "эмоциональное истощение"
]
```

**Объекты (legacy):**
```json
"markers": [
  { "key": "resilience", "label": "Устойчивость", "value": "высокая" }
]
```

Публичная страница корректно отображает оба формата.

---

### `report.layers[]`

| Поле | Тип | Описание |
|---|---|---|
| `name` | string | Название уровня (например: «Потребности», «Эмоции») |
| `description` | string | Описание уровня анализа |

---

### `meta`

| Поле | Тип | Описание |
|---|---|---|
| `meta.schema_version` | string | Версия схемы: `"1.0"` |

---

## Отображение на публичной странице `/r/:token`

| Элемент | Что отображается |
|---|---|
| Hero-карточка | `report.title`, `participant.name`, мета-чипы (язык, источник, версия, дата) |
| Краткое резюме | `report.summary` |
| Ключевые маркеры | `report.markers[]` как цветные чипы |
| Блоки анализа | Каждый `block` — отдельная карточка с `title`, `text`, `quotes`, `recommendations` |
| Уровни анализа | `report.layers[]` как сетка карточек |
| Дисклеймер | Всегда: «Это автоматический аналитический отчёт...» |

---

## Fallback для нестандартной структуры

Если JSON не содержит поле `report` на верхнем уровне — страница показывает:

> Отчёт получен и сохранён.  
> Структура отчёта отличается от стандартной Mindloom schema v1.

Raw JSON на публичной странице **не показывается никогда**.
